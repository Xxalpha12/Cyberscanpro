What is TAM?

The Technology Acceptance Model (TAM), developed by Fred Davis in 1989, explains how users come to accept and use a technology system. It identifies two primary factors that determine whether someone will use a system:

1.Perceived Usefulness (PU) — the degree to which a user believes the system will enhance their work performance. In your project: does CyberScan Pro make vulnerability assessment more efficient?
2.Perceived Ease of Use (PEOU) — the degree to which a user believes the system will be free of effort. In your project: is the scanning interface intuitive enough for security practitioners to adopt?
// TAM DIAGRAM
External Variables ──→ Perceived Usefulness (PU) ──→ Behavioural Intention ──→ Actual Use └──────────────→ Perceived Ease of Use (PEOU) ─────────────────────────────↑ └──────────────────────────────────────→ PU

Why TAM for CyberScan Pro?

CyberScan Pro is a technical tool built for security professionals and organisations. TAM is appropriate because the project aims to develop a system that is both functionally effective (high PU) and practically usable without advanced configuration (high PEOU). The success of the tool depends on whether users will actually adopt it over commercial alternatives.

Key answer for defense: "I adopted the Technology Acceptance Model (TAM) by Davis (1989) because my research evaluates not just whether CyberScan Pro can detect vulnerabilities, but whether it is practically useful and easy enough for real-world adoption by IT security teams and organisations."

What is Empirical Review?

An empirical review presents findings from existing research studies, experiments, and surveys that are directly relevant to your project. Unlike a conceptual review (which discusses theories and concepts), an empirical review focuses on what other researchers have actually tested, measured, and concluded.

Relevant Empirical Studies for CyberScan Pro

1.Nessus vs Open-Source Scanners (Gordon et al., 2020) — Found that open-source tools like OpenVAS detected 78% of vulnerabilities found by commercial tools, but generated 34% more false positives. This supports your rationale for building an improved automated tool.
2.Web Application Vulnerability Prevalence (OWASP, 2021) — The OWASP Top 10 report found SQL Injection, XSS, and Broken Authentication remain the three most critical web vulnerabilities, appearing in over 60% of tested web applications. CyberScan Pro specifically targets these.
3.CVE Disclosure Impact (NVD/NIST, 2022) — Studies show the average time between CVE publication and active exploitation is now under 7 days. Automated scanning tools that cross-reference live CVE databases are therefore essential.
4.SME Cybersecurity Posture (Ponemon Institute, 2022) — 68% of small-to-medium enterprises do not conduct regular vulnerability assessments due to cost and complexity. This validates the need for an accessible, automated tool like CyberScan Pro.
Key answer for defense: "The empirical review examines existing studies on vulnerability assessment tools, penetration testing methodologies, and web security frameworks. Evidence from OWASP (2021) and NVD/NIST (2022) confirms the persistent prevalence of the vulnerabilities my tool targets, while Ponemon Institute (2022) validates the gap in accessible automated scanning solutions."

Research Design
Design type, population, sampling, data collection
▼

Research Design Type

CyberScan Pro uses a Design and Development Research (DDR) methodology, also known as Design Science Research. This is appropriate because the primary output of the study is a working software artefact (the scanning system) rather than a survey or experiment.

1.Problem Identification — Lack of accessible, affordable automated vulnerability assessment tools for Nigerian institutions.
2.Design and Development — Building CyberScan Pro using Python/Flask, integrating NVD, network scanning, and report generation.
3.Evaluation — Testing the system against known vulnerable targets and comparing findings against established tools.
4.Communication — Documenting results in this project report and defending findings.
// RESEARCH DESIGN FLOW
Problem Statement ↓ Literature Review (Conceptual + Empirical) ↓ System Design (Architecture, Database Schema) ↓ System Development (Python/Flask, PostgreSQL) ↓ System Testing (Functional + Vulnerability Tests) ↓ Results Analysis & Report Generation ↓ Conclusion & Recommendations

Population of Study

The population of study consists of IT security professionals, system administrators, and network engineers within Nigerian tertiary institutions and SMEs who are responsible for managing and securing digital infrastructure. FUPRE's internal network and web systems serve as the primary test environment.

Sampling Technique

A purposive (judgement) sampling technique was used. This means participants or test environments were deliberately selected based on their relevance to the research objectives — specifically, systems with known network exposure and web applications accessible for ethical testing. This is appropriate for technical research where random sampling would not guarantee relevant system characteristics.

Data Collection Method

1.Primary Data — Collected through the system itself. Each scan generates structured data: open ports, detected services, web vulnerabilities, and CVE mappings stored in a PostgreSQL database.
2.Secondary Data — From NVD/NIST CVE database, OWASP documentation, and existing vulnerability assessment literature.
3.Questionnaire — A structured questionnaire using a 5-point Likert scale was administered to IT professionals to evaluate the system's usability (PEOU) and effectiveness (PU), aligned with the TAM framework.

Data Analysis Method

Both quantitative and qualitative analysis methods were employed. Quantitative analysis involved statistical tabulation of questionnaire responses using frequency tables and percentage scores. Qualitative analysis involved comparing CyberScan Pro's findings against manually verified vulnerabilities to calculate detection accuracy and false positive rates.

The Problem

Despite the growing sophistication of cyber threats, many Nigerian institutions and small-to-medium enterprises lack the resources to deploy commercial vulnerability assessment tools such as Nessus or Qualys. Existing open-source alternatives, while functional, often require significant technical expertise to configure and operate, limiting their adoption.

Furthermore, most available tools do not provide contextual, plain-English vulnerability reports with actionable remediation guidance — a critical requirement for non-specialist stakeholders such as management and compliance officers. The absence of an accessible, integrated, and intelligible vulnerability assessment platform contributes directly to the persistently poor cybersecurity posture of many Nigerian organisations.

Core Problem Statement: The lack of an affordable, accessible, and automated web-based vulnerability assessment platform that integrates network scanning, web vulnerability testing, CVE intelligence, and professional report generation represents a critical gap in cybersecurity tooling available to Nigerian institutions.

How CyberScan Pro Solves This

1.Provides a web-based interface eliminating the need for command-line expertise.
2.Integrates NVD for real-time CVE intelligence without manual lookup.
3.Generates professional PDF and HTML reports with plain English explanations.
4.Operates at zero licensing cost, making it viable for resource-constrained institutions.

Research Gap
The flaw in previous work your project solves
▼

Identified Research Gaps

1.Lack of Integration: Existing tools such as OpenVAS, Nikto, and Nmap operate as standalone utilities. No previous study in the Nigerian context has presented a single integrated platform combining network scanning, web vulnerability testing, and CVE intelligence with automated report generation.
2.Accessibility Gap: Prior tools require command-line proficiency and complex installation procedures, making them inaccessible to non-specialist IT staff. CyberScan Pro addresses this through a browser-based interface deployable without local installation.
3.Reporting Inadequacy: Existing tools produce technical output (XML, JSON, terminal text) that is unsuitable for management-level reporting. CyberScan Pro generates professional PDF and HTML reports with plain English explanations and remediation guidance.
Primary gap to emphasise in defense: No existing open-source or academic tool provides a fully web-based, integrated vulnerability assessment platform with professional reporting capabilities suitable for deployment in Nigerian institutional environments without commercial licensing.
